# 📊 Retail AI Suite — Model Evaluation Metrics Report

This document compiles the actual, empirically-measured evaluation metrics, feature spaces, and verification results of the machine learning and analytical models integrated in the Retail AI Suite platform, generated by executing the training and validation pipelines on the raw datasets on a CUDA-enabled GPU (RTX 3050).

---

## 1. 🔮 Customer Churn Prediction
* Model Type: Supervised Binary Classification
* Algorithms: Scikit-Learn Random Forest Classifier (CPU fallback tuning for balanced class weights)
* Objective: Predict if a customer is at risk of churn based on their engineered transaction history and demographic profiles.
* Dataset Size: 250,000 transaction rows (grouped into 39,920 unique customers; split into 23,952 Train, 7,984 Validation, 7,984 Test).

### Feature Space (Inputs)
* Demographics: Gender, Age
* Recency: days_since_last_purchase (Recency Risk)
* Frequency: total_orders (Purchase Frequency), frequency_score
* Monetary Value: total_revenue, avg_order_value, monetary_score
* Interaction: unique_categories (Category Diversity), total_returns, return_rate

### Actual Evaluation Performance Metrics
Measured on the Test set (Stratified Customer-level Split):

| Metric                  |   Score  | Interpretation / Notes                                                    |
| :---------------------- | :------: | :------------------------------------------------------------------------ |
| Accuracy                |  71.18%  | Overall correct predictions rate                                          |
| Precision               |  35.35%  | Out of predicted churners, the % who actually churned                     |
| Recall (Sensitivity)    |  28.14%  | Out of actual churners, the % correctly flagged by the model              |
| F1-Score                |  31.33%  | Harmonic mean of precision and recall                                     |
| Specificity             |  84.31%  | True negative rate (correctly identified safe customers)                  |
| Balanced Accuracy       |  56.22%  | Balanced score adjusting for class imbalance (test positive rate: 23.37%) |

| PR-AUC                  |  34.06%  | Area under Precision-Recall curve                                         |

### Actual Confusion Matrix

|                  |     Predicted Safe        |     Predicted Churn       |
| :--------------- | :-----------------------: | :-----------------------: |
| Actual Safe      |  5,158 (True Negative)    |  960 (False Positive)     |
| Actual Churn     |  1,341 (False Negative)   |  525 (True Positive)      |

* TN (True Negatives): Correctly predicted active/safe customers.
* TP (True Positives): Correctly identified at-risk customers.
* FP (False Positives): Active customers incorrectly labeled as at-risk (marketing buffer).
* FN (False Negatives): Customers who churned without warning.

### Top Churn Feature Importances

2. Frequency Score (frequency_score): 7.16%
3. Total Revenue (total_revenue): 5.36%
4. Monetary Score (monetary_score): 5.01%

* Algorithms: ARIMA (Auto-Regressive Integrated Moving Average) order=(1, 1, 0)
